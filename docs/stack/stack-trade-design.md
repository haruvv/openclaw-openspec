# AIトレード育成WebApp 構想まとめ

## 目的

実際のお金をいきなり動かすのではなく、まずは実際の株価・チャート・ニュース・決算情報を使ったペーパートレード環境を作る。

その上で、AIに売買判断をさせ、結果を記録し、反省・改善・知識化を繰り返すことで、株取引用AIエージェントとSkillsを育てる。

最終的には、以下をWebApp上で見えるようにする。

- 与えた資金がどう動いているか
- AIが何を買ったか / 売ったか
- AIがなぜその判断をしたか
- テクニカル / ファンダ / ニュース / 決算をどう見たか
- 取引結果から何を学んだか
- 次回の判断ルールがどう変わったか

## 基本方針

このアプリは、証券アプリそのものを作るのではなく、AIトレーダーの運用ダッシュボードとして作る。

裏側では moomoo API や TradingView、既存のペーパートレード環境を使う。

WebApp側では、AIの判断・取引ログ・学習ログ・資産推移を管理する。

構成イメージ:

- moomoo / TradingView / 外部データ
- 自分のWebApp
- AI Decision Engine
- Paper Trade / Demo Trade
- Trade Log DB
- Skills / Knowledge Base

## 投資スタイル

長期投資に寄せすぎると、投資信託やETFと差別化できない。

そのため、狙う時間軸は以下が良さそう。

- 数日〜数週間
- 長くても1〜3ヶ月程度

### 差別化ポイント

投資信託 / ETF:

- 市場全体に乗る
- 長期でじわじわ増やす
- 基本的に買って持ち続ける

AIトレードWebApp:

- 今強いテーマや銘柄を選ぶ
- 決算・ニュース・テクニカルを見て判断する
- 悪くなったら早めに撤退する
- 取引理由と学習ログを残す
- 戦略ごとの成績を可視化する

## 狙う戦略

### 1. イベントドリブン短期スイング

決算、上方修正、自社株買い、大型受注、政策テーマ、セクター資金流入などをきっかけに、数日〜数週間で値幅を狙う。

対象イベント:

- 決算
- 上方修正
- 自社株買い
- 大型受注
- 政策テーマ
- セクター資金流入
- 同業連れ高

保有期間:

- 3日〜20営業日程度

判断:

- ファンダ / ニュースで銘柄選定
- テクニカルでエントリー
- 損切り / 利確はルール固定
- 結果をAIがレビュー

### 2. 決算後モメンタム

好決算銘柄をすぐ長期保有するのではなく、短期〜中期で市場の反応を取りに行く。

見るポイント:

- 決算内容が良い
- 市場もポジティブに反応している
- 出来高が増えている
- 高値掴みではない
- 数日〜数週間の上昇余地がある

### 3. テーマローテーション

AI、半導体、電線、防衛、ロボット、データセンター、金融など、今資金が来ているテーマに乗る。

見るポイント:

- 今週強いセクター
- 出来高が増えている業界
- ニュースで注目されているテーマ
- 関連銘柄の連れ高

### 4. 押し目買い・過熱回避

良い銘柄でも、急騰直後には買わない。

条件:

- 良い銘柄
- 急騰直後ではない
- 押し目形成中
- 損切り位置が明確

この条件を満たすときだけエントリー候補にする。

## moomoo APIの位置づけ

moomoo APIは、株価・チャート・リアルタイム情報・過去ローソク足などのデータソースとして使える。

自分で最初から市場データを大量に用意する必要はなく、まずは moomoo API をメインデータソースにする。

ただし、取得したデータは自分のDBにも保存する。

構成:

- moomoo API
- Market Data Collector
- 自分のDB
- AI分析 / バックテスト / WebApp表示

### moomoo APIから取れそうなもの

- リアルタイム株価
- 過去ローソク足
- 直近ローソク足
- ティック
- 板情報
- 資金フロー
- セクター情報
- 銘柄情報

### moomooだけでは足りない可能性があるもの

- 決算短信
- 有価証券報告書
- 適時開示
- ニュース本文
- X / YouTube / 掲示板などの市場センチメント
- AI判断ログ
- 学習ログ
- 自分専用の取引ルール

そのため、最終的には以下のように分ける。

価格データ:

- moomoo API

決算 / 開示:

- TDnet
- EDINET
- 企業IR
- ニュースAPI

判断ログ:

- 自分のWebApp

学習ログ:

- 自分のDB
- Skills repo

## AIエージェント構成

10体程度の株取引用エージェントを用意し、それぞれ判断材料と責任範囲を分ける。

重要なのは、全員を独立したトレーダーにするのではなく、専門家エージェントたちで1つの投資判断を作ること。

## エージェント一覧

### 1. Market Scanner Agent

市場全体・セクター・テーマを監視する。

判断材料:

- 日経平均
- TOPIX
- NASDAQ
- S&P500
- セクター別騰落
- 値上がり率
- 出来高急増
- テーマ株の資金流入
- 市場全体のリスクオン / リスクオフ

出す結論:

- 今どこに資金が来ているか
- どのテーマを優先監視すべきか
- 今日は攻めるべきか、守るべきか

### 2. Fundamental Agent

企業の業績・バリュエーション・成長性を見る。

判断材料:

- 決算
- 売上
- 営業利益
- 純利益
- EPS
- PER
- PBR
- ROE
- 成長率
- 業績予想
- 上方修正 / 下方修正
- 配当
- 自社株買い

出す結論:

- その銘柄を触る価値があるか
- 中期的に買われる理由があるか
- 業績面でポジティブかネガティブか

### 3. News Agent

ニュース・適時開示・材料の新規性を見る。

判断材料:

- ニュース
- TDnet適時開示
- 企業IR
- 政策ニュース
- 業界ニュース
- 同業他社ニュース
- 為替 / 金利 / マクロ材料
- 材料発表後の株価反応

出す結論:

- 材料は強いか
- 既に織り込まれているか
- 短期的に買われる理由があるか
- 悪材料はないか

### 4. Technical Agent

チャートと需給を見る。

判断材料:

- ローソク足
- 移動平均線
- 出来高
- 高値更新
- 安値割れ
- 支持線 / 抵抗線
- RSI
- MACD
- VWAP
- トレンドライン
- ギャップアップ / ギャップダウン

出す結論:

- 今入れる形か
- 押し目待ちか
- 過熱しているか
- トレンドが崩れているか

### 5. Entry Agent

具体的にどこで入るかを判断する。

判断材料:

- 押し目
- ブレイクアウト
- 支持線
- 直近高値
- 出来高
- 指値候補
- 損切り位置との距離

出す結論:

- 成行で入るか
- 指値で待つか
- どの価格帯なら入ってよいか
- 今は見送るべきか

### 6. Exit Agent

どこで利確・損切り・撤退するかを判断する。

判断材料:

- 利確目標
- 損切りライン
- トレンド崩れ
- 出来高減少
- 材料出尽くし
- 抵抗線到達
- 保有期間
- 時間切れ

出す結論:

- 利確するか
- 損切りするか
- 半分利確するか
- 継続保有するか

### 7. Risk Manager Agent

リスク管理を担当する。拒否権を持たせる。

判断材料:

- 資金量
- 1回あたりの損失許容
- 損切り幅
- リスクリワード
- 連敗数
- 最大ドローダウン
- 銘柄のボラティリティ
- 板の薄さ
- セクター集中
- 決算前後リスク

出す結論:

- その取引を許可するか
- ポジションサイズはいくらか
- 損切り幅が広すぎないか
- 見送るべきか

### 8. Portfolio Agent

資金配分・保有銘柄全体を見る。

判断材料:

- 現金比率
- 保有銘柄
- セクター偏り
- 含み損益
- 確定損益
- 相関
- 同一テーマへの集中
- 余力

出す結論:

- 追加で買ってよいか
- 既存ポジションを減らすべきか
- 特定テーマに偏りすぎていないか
- 現金を残すべきか

### 9. Review / Learning Agent

取引後に勝因・敗因を分析する。

判断材料:

- エントリー時の判断ログ
- 決済時の判断ログ
- 実際の損益
- 最大含み益
- 最大含み損
- 保有中のニュース変化
- 各Agentの当時の意見

出す結論:

- 何が良かったか
- 何が悪かったか
- どのAgentの判断が正しかったか
- どのAgentの判断が間違っていたか
- 次回改善すべき点

### 10. Knowledge Curator Agent

取引レビューから、再利用可能な知識・ルール・Skill改善案を抽出する。

判断材料:

- Review Agentの分析
- 過去の類似取引
- 戦略別成績
- 負けパターン
- 勝ちパターン
- 禁止ルール

出す結論:

- 新しいルール候補
- 禁止ルール
- Skillに反映すべき内容
- 既存ルールの修正案
- 採用 / 保留 / 却下

### 11. Judge Agent

各Agentの意見を統合して最終判断する。

判断材料:

- Market Scannerの意見
- Fundamentalの意見
- Newsの意見
- Technicalの意見
- Entryの意見
- Exitの意見
- Risk Managerの意見
- Portfolioの意見

出す結論:

- BUY
- SELL
- HOLD
- WATCH
- SKIP

ただし、Risk Manager Agent が拒否した場合は、Judge Agent が買い推奨でも取引しない。

## 判断フロー

1. Market Scanner が市場とテーマを確認
2. 候補銘柄を抽出
3. Fundamental Agent が業績面を評価
4. News Agent が材料とニュースを評価
5. Technical Agent がチャートを評価
6. Entry Agent が入る価格帯を提案
7. Risk Manager Agent がリスクを審査
8. Portfolio Agent が資金配分を確認
9. Judge Agent が総合判断
10. デモ取引 / ペーパートレード実行
11. Exit Agent が決済判断
12. Review Agent が結果を分析
13. Knowledge Curator Agent が知識化
14. Skills / Knowledge Base に反映

## Agent同士の意見が割れる例

Fundamental Agent:

- 業績成長期待と半導体需要は強い
- 中期ではポジティブ

News Agent:

- 直近材料はポジティブ
- ただし株価には一部織り込み済み

Technical Agent:

- 短期チャートは強い
- ただし急騰後なので今すぐ買うのは危険

Risk Manager:

- 損切り位置が遠い
- 今入るとリスクリワードが悪い

Judge Agent:

- 結論、監視継続
- 押し目形成まで買わない

このように、意見が割れること自体に価値がある。

## Risk Managerの拒否権

Risk Manager Agent には拒否権を持たせる。

例:

Judge Agent:

- 買い推奨

Risk Manager:

- 拒否
- 損切り幅が8%あり、想定利益3%に対してリスクが大きすぎる

Final:

- 見送り

これにより、AIが雰囲気で買うことを防ぐ。

## バックテストと過去チャート学習

今後の取引ログだけでなく、過去のチャート値動きからも学習させる。

やることは大きく2つ。

### 1. バックテスト

過去の株価データで「このルールなら勝てたか」を検証する。

### 2. パターン学習

過去の値動きから「勝ちやすい形・負けやすい形」を知識化する。

### バックテストで見る指標

- 取引回数
- 勝率
- 平均利益
- 平均損失
- 期待値
- Profit Factor
- 最大ドローダウン
- 最大連敗
- 銘柄別成績
- 戦略別成績
- 相場環境別成績

### 過去検証で注意すること

- その時点で分かっていた情報だけ使う
- 未来の決算や株価を見ない
- 手数料を入れる
- スプレッドを入れる
- スリッページを入れる
- 約定ズレを入れる
- 条件をいじりすぎない
- 開発に使っていない期間で検証する
- 特定銘柄だけで勝っていないか確認する

## 本番移行の最低条件

バックテスト:

- 取引回数 300回以上
- 期待値プラス
- Profit Factor 1.3以上
- 最大ドローダウン -15%以内
- 特定の1銘柄だけで勝っていない
- 直近未使用期間でもプラス

ペーパートレード:

- 最低2〜3ヶ月
- 取引回数 50回以上
- 手数料 / スリッページ込みでプラス
- 最大連敗を想定内に収める
- ルール違反時に自動停止できる

実弾テスト:

- 最初は5万円以下
- 1回の損失は資金の1〜2%以内
- 月間損失上限を決める
- 連敗したら停止

## 期待値

勝率だけではなく、期待値を見る。

期待値:

- 勝率 × 平均利益 - 負け率 × 平均損失

勝率が高くても、負ける時に大きく負けるならダメ。

勝率が低くても、利益が損失より大きければ勝てる可能性がある。

## この構想で一番価値がある部分

「AIが相場を当てる」ことよりも、以下の方が価値がある。

- AIが悪い取引を止める
- AIが根拠の薄いエントリーを弾く
- AIが過去の失敗パターンを警告する
- AIが損切りラインを守らせる
- AIが戦略ごとの期待値を評価する

人間の負け原因は予測ミスだけではない。

よくある負け原因:

- 焦って買う
- 損切りできない
- 利確が雑
- 上がった理由を後付けする
- 負けを取り返そうとする
- 検証していないルールで入る
- 急騰後に飛び乗る
- 材料を過大評価する
- リスクリワードを見ない

これをAIとWebAppで矯正する。

## Skills構成案

skills:

- market-scan
- fundamental-analysis
- news-analysis
- technical-analysis
- entry-check
- exit-check
- risk-management
- portfolio-management
- trade-review
- knowledge-extract
- judge

### market-scan

- 今日の強いテーマを探す
- 出来高急増銘柄を整理する
- セクター資金流入を確認する
- 監視候補を出す

### fundamental-analysis

- 決算を読む
- 業績成長を見る
- PER / PBRを見る
- 上方修正 / 下方修正を見る
- 中期的に買われる理由があるか判断する

### news-analysis

- ニュースを読む
- 適時開示を読む
- 材料の新規性を見る
- 既に織り込まれているか判断する

### technical-analysis

- チャートを見る
- 出来高を見る
- 移動平均線を見る
- 押し目 / ブレイク / 過熱を判断する

### entry-check

- エントリーしてよいか確認する
- 損切り位置が明確か確認する
- リスクリワードを計算する
- 過去の失敗パターンに該当しないか見る

### exit-check

- 利確すべきか判断する
- 損切りすべきか判断する
- トレンド崩れを確認する
- 保有継続か撤退か判断する

### risk-management

- ポジションサイズを決める
- 最大損失を制御する
- 連敗時に停止する
- Risk Managerの拒否権を実装する

### trade-review

- 取引結果を分析する
- 勝因 / 敗因を分類する
- Agentごとの判断が正しかったか評価する
- 改善点を出す

### knowledge-extract

- 取引レビューから再利用可能なルールを抽出する
- 勝ちパターン / 負けパターンを保存する
- 禁止ルールを作る
- Skillsに反映する候補を出す

### judge

- 各Agentの意見を統合する
- 最終判断を出す
- Risk Managerの拒否がある場合は取引を止める

## 最終コンセプト

このWebAppは、普通の取引アプリではない。

コンセプト:

AIに資金を預けたら、
何を考えて、
何を買って、
なぜ売って、
結果から何を学んだのかを可視化するアプリ。

目指すべきは、完全自動で爆益を出すBotではなく、AIが短期〜中期のテーマ株を選び、決算・ニュース・テクニカルを根拠に売買し、結果から運用ルールを育てるアプリ。

最初の成功条件は「AIが相場を当てる」ことではない。

成功条件:

- 根拠の薄い取引を減らす
- 悪いエントリーを止める
- 損切りを守る
- 期待値の悪い戦略を捨てる
- 学習ログを蓄積する
- 戦略ごとの成績を可視化する

ここまでできれば、少額で試す価値のあるAIトレード育成環境になる。
