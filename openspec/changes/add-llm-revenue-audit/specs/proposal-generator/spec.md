## MODIFIED Requirements

### Requirement: Lighthouse診断結果をもとにSEO改善提案書を生成する
システムは、サイトクローラーが出力したターゲットリスト（URL・スコア・診断項目）と、利用可能な場合はLLM営業評価を入力として受け取り、`llm-provider` モジュール経由で LLM を呼び出して企業ごとにパーソナライズされたSEO改善提案書をMarkdown形式で生成しなければならない（SHALL）。使用する LLM プロバイダーは `llm-provider` が決定し、`proposal-generator` はプロバイダーを意識しない。

#### Scenario: 提案書の生成に成功する
- **WHEN** ターゲット企業のURL・SEOスコア・診断項目がシステムに渡される
- **THEN** 企業名・業種・具体的な改善箇所を含むMarkdown提案書が生成される

#### Scenario: LLM営業評価を使って提案書を生成する
- **WHEN** ターゲット企業の調査結果に `llmRevenueAudit` が含まれる
- **THEN** 提案書は営業優先度、事業影響、推奨オファー、注意事項を反映する
- **AND** 初回接触文面は返信獲得を目的とした丁寧で非断定的な表現になる

#### Scenario: 診断項目が不足している場合
- **WHEN** Lighthouse診断結果が1項目しか含まれていない
- **THEN** 利用可能な情報のみで提案書を生成し、「詳細情報不足」の注記を追加する

#### Scenario: 営業評価が存在しない場合
- **WHEN** `llmRevenueAudit` が存在しない
- **THEN** システムは既存のSEOスコア、診断項目、改善余地findingを使って提案書を生成する

## ADDED Requirements

### Requirement: 初回接触文面は敬意ある無料診断共有として生成する
システムは、初回接触向けの文面を生成する場合、売り込みや断定的な指摘ではなく、無料の簡易診断共有として返信を促す文面を生成しなければならない（SHALL）。

#### Scenario: 初回接触文面を生成する
- **WHEN** 提案書または営業評価に初回メール文面が含まれる
- **THEN** 文面は「確認した範囲では」「もし必要であれば」などの非断定的な表現を使う
- **AND** 売上向上の保証、相手を責める表現、即時購入を迫る表現を含まない
